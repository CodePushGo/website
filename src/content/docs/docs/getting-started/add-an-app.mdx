## Requirements

Before getting started with CodePushGo, make sure you have:

- A React Native app with an iOS bundle identifier or Android application ID
- Node.js 18 or later
- A CodePushGo account at [https://codepushgo.com/register/](https://codepushgo.com/register/)

<Aside type="note">
Registration creates the account only. Plan intent is recorded inside console onboarding, after the account exists.
</Aside>

## Guided setup

<Steps>

1. Open the console at [https://console.codepushgo.com/app/home](https://console.codepushgo.com/app/home).

2. Log in with the CLI from your React Native project folder.

   <Code code={`npx @codepushgo/cli@latest login`} lang="bash" />

3. Connect the app. By default the CLI detects the native app identity from Expo config, iOS `PRODUCT_BUNDLE_IDENTIFIER`, or Android `applicationId`.

   <Code code={`npx @codepushgo/cli@latest init`} lang="bash" />

4. Install the updater client.

   <Code code={`npm install @codepushgo/react-native-updater`} lang="bash" />

5. Deploy a live update.

   <LinkCard href="/docs/getting-started/deploy/" title="Deploy a live update" description="Build and upload your first React Native JavaScript bundle." />

</Steps>

## Manual setup

Use manual setup only when the CLI cannot infer the app identity.

<Steps>

1. Create or update the app using its native bundle ID.

   <Code code={`npx @codepushgo/cli@latest app add com.example.app --name "Example App"`} lang="bash" />

2. Create a local config file.

   <Code code={`{\n  "appId": "com.example.app",\n  "channel": "production",\n  "endpoint": "https://api.codepushgo.com"\n}`} lang="json" />

3. Configure the JavaScript updater in your app startup code.

   <Code code={`import { configureCodePushGo, startCodePushGo } from '@codepushgo/react-native-updater'\n\nconfigureCodePushGo({ endpoint: 'https://api.codepushgo.com' })\nawait startCodePushGo({ defaultChannel: 'production' })`} lang="typescript" />

4. Upload your first release.

   <Code code={`npx @codepushgo/cli@latest release --platform ios --version 1.0.1 --channel production`} lang="bash" />

</Steps>
